## 案例1：mac迁移数据到另一台mac

时间：20180518

debug用时：8小时

1）[How to fix homebrew permissions?](https://stackoverflow.com/questions/16432071/how-to-fix-homebrew-permissions) brew permission denied？

```
sudo chown -R "$USER":admin /usr/local（路径）
```

2）brew检测

```
brew missing
brew doctor
```

3）报错

```
/Users/parkerharris/.rvm/rubies/ruby-2.3.0/lib/ruby/2.3.0/irb/completion.rb:10:in `require': dlopen(/Users/parkerharris/.rvm/rubies/ruby-2.3.0/lib/ruby/2.3.0/x86_64-darwin15/readline.bundle, 9): Library not loaded: /usr/local/opt/readline/lib/libreadline.6.dylib (LoadError)
  Referenced from: /Users/parkerharris/.rvm/rubies/ruby-2.3.0/lib/ruby/2.3.0/x86_64-darwin15/readline.bundle

```

解决办法：搜索rails c `require': dlopen

https://stackoverflow.com/questions/39885288/unable-to-execute-rails-console-command-ruby

```
brew cleanup --prune=30
brew update
brew upgrade
```

4）报错：搜索技巧——要搜not loaded的内容

```
`const_missing': library not found for class Digest::SHA1 -- digest/sha1 (LoadError): Library not loaded: /opt/local/lib/libssl.1.0.0.dylib (LoadError)
```

解决办法：

①、搜索“ Library not loaded: /usr/local/opt/readline/lib/libreadline.7.dylib (LoadError) ”

```
brew rm -f readline
brew install readline
brew link readline --force
```

②、搜索“ Library not loaded: /usr/local/opt/openssl/lib/libssl.1.0.0.dylib (LoadError) ”

```
brew remove openssl 
brew install openssl
```

### 总结：

搜到很多解答，大多类似于@[PPL](https://stackoverflow.com/users/9411789/ppl)的解决办法，但是这个对我没用。最后，我通过brew upgrade找到问题所在是openssl，接着是readline、nikogiri、dylib等的问题。

需要特别注意的是，报错的解决办法常常不是重装报错信息里提到的gem，而是重装与它关联的其他gem。比如我的报错信息出现dylib，结果却是dylib所依赖的node的问题，重装node后解决。报错信息出现bindex，结果却是bindex所依赖的libiconv的问题，重装libiconv后解决。

最后发现，以上问题都是由dmg外部安装而不是homebrew安装的curl和node引发的。
